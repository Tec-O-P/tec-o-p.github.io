Switches

Observações

(O que está em vermelho não está funcionando pois tal funcionalidade está oculta no código.)

  • Não altere nada a não ser que seja absolutamente necessário! Para evitar ter que refazer 1 trilhão de testes!
  • Todas as amostras de uso possuem a mesma estrutura semântica, exceto as que estão na parte de testes 'Switches Sem Peças Desnecessárias No Código'.
  • Ao estilizar os switches, coloque os estilos na folha de estilos CSS (com !important) ao invés de colocá-los inline, pois os estilos das funções que atualizam os switches sobrepõem alguns estilos inline. (Também por este motivo, alguns estilos dos exemplos abaixo foram colocados na folha de estilos CSS.)
  • O tamanho dos ícones dos switches são configurados por padrão com 32 x 32 no CSS, exceto os de tipo Image que são 48 x 48, logo, caso queira um tamanho diferente desses, deve-se configurá-los manualmente.
  • O preenchimento de um switch de botão de ícone (switch com a classe 'button_icon') é definido automaticamente proporcionalmente ao tamanho do ícone.
  • Os preenchimentos dos switches de tipo button's são definidos automaticamente relativos aos tamanhos de seus conteúdos.

Notas sobre bugs

  • As sombras internas criadas pelo box-shadow ('box-shadow: inset ...') dos switches dos tipos 'button_full', 'button_text', e 'button_icon' com qualquer uma das classes 'pill' ou 'rounded' PODEM não renderizar corretamente no Safari para desktop dependendo do tamanho da janela do navegador (seja o tamanho horizontal ou vertical). Para resolver isso, tive que fazer com que os que possuem as classes 'pill' e 'rounded' percam estas classes (ou seja, ficarão quadrados) no Safari para desktop e nos navegadores de tablets.
  • Os switches com as classes 'button_icon' e 'rounded' ficam muito grandes no mobile. Para resolver isso, substitui neles a classe 'rounded' por 'pill' no mobile.
  • Switches com as classes 'button_icon' e 'pill', e com o ícone com a altura muito maior do que a largura, PODEM ficar com o brilho de cima muito estreito.
